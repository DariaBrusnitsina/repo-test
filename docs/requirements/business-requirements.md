# Бизнес-требования

## Цель

Предоставить витрине простой механизм расчёта итоговой суммы заказа без подключения тяжёлой логики и дополнительных зависимостей.

## Перечень требований

| ID | Требование | Приоритет |
| --- | --- | --- |
| BR-01 | Система должна рассчитывать сумму по формуле `цена * количество + доставка` | Высокий |
| BR-02 | Система должна определять признак бесплатной доставки | Средний |
| BR-03 | Расчёт должен выполняться без обращения к внешним системам | Высокий |
| BR-04 | Логика должна быть понятной для демонстрации и обучения | Высокий |

## Ограничения

1. Решение не должно требовать сторонних библиотек.
2. Код должен быть компактным и читаемым.
3. Документация должна храниться в формате Markdown.

## PlantUML-диаграмма требований

```plantuml
@startuml
left to right direction

actor "Покупатель" as Customer
actor "Витрина" as UI

rectangle "Сервис расчёта" {
  usecase "Рассчитать сумму заказа" as UC1
  usecase "Определить бесплатную\nдоставку" as UC2
}

Customer --> UI
UI --> UC1
UI --> UC2
UC2 ..> UC1 : использует результат
@enduml
```
